  - Root cause: race condition in alpha build's arm+takeoff command sequence
  - Fix: downgraded to PX4 v1.15.4 (stable) + launch Gazebo standalone before PX4 in standalone mode

```

```

- Demo video and installation guide: pending final recording

### Task: Real-time log generation
- PX4 flight logger (`.ulg` files): working via `logger start`
- Live log monitoring commands set up (watch, tail -f)
- MAVLink packet capture (Wireshark): working
  - Key finding: WSL2 routes true loopback traffic through interface **`loopback0`**, not `lo`
  - MAVLink protocol dissector (`mavlink_common.lua`) installed to `~/.local/lib/wireshark/plugins/`
  - Filter `mavlink_proto` or `udp.port == 14550` on `loopback0` shows decoded MAVLink messages

### MAVLink v1/v2 switching (for study/comparison)
- Parameter: `MAV_PROTO_VER` (0=auto, 1=force v1, 2=force v2)
- **Important**: live-switching via `mavlink stop-all` / `mavlink start` caused connection issues —
  safer to set the parameter then do a full relaunch instead
- Confirmed: GCS link auto-negotiates to v2 when QGC connects; unused links (companion/gimbal) stay v1 by default since nothing negotiates with them — this is expected behavior, not a bug
